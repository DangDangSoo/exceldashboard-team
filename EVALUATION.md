# EVALUATION.md — Excel2Dashboard Pro 자체평가

| 평가축 | 판정 | 근거 |
|---|---|---|

## 테스트 방법 메모

- **영속성/삭제**: curl로 데이터셋 업로드 → 태그 부여 → 열 타입 교정 → 분석 저장(차트/집계/피벗 각 1개 이상) → 서버 프로세스 재기동 → 재호출·재현 확인. 삭제는 `sqlite3 data/app.db`로 3테이블 행 수를 직접 세고, `ls uploads/{id}/`로 디스크 상태를 직접 확인.
- **정확성**: `pandas`로 동일 계산을 직접 수행해 API 응답과 diff(Basic 때와 동일한 방법 유지).
- **회귀**: Day1 완료 후 Day2 작업, Day2 완료 후 Day3 작업을 시작하기 전/후 각각 기존 curl 스위트를 재실행해 이상 없음 확인.
- **UI**: 실제 브라우저에서 업로드→태그→재호출→분석저장→보기→삭제 전체 흐름을 조작해 콘솔 에러 없음 확인. (참고: 이번 세션 도구 환경에서 브라우저 클릭 자동화 도구가 간헐적으로 좌표를 잘못 잡는 이슈가 있어, 동일한 결과를 내는 JS `element.click()`으로 우회 검증한 구간이 있음 — 앱 자체 문제 아님.)

## 알려진 한계 (다음에 고려할 것)

- **동시성·장기구동 미검증**: Basic 때와 동일하게 여전히 확인 안 함. Pro는 SQLite를 쓰므로 동시 쓰기에 특히 약하지만, 단일 사용자 전제라 설계상 우선순위가 낮다고 판단(Team 단계에서 재검토 필요).
- **디스크 용량 자동 정리 없음**: 삭제 API로 수동 정리는 가능하지만, 업로드가 계속 쌓여도 자동으로 지워지지 않음(PRD에서 의도적으로 범위 밖으로 뺀 부분).
- **피벗/집계 PNG 가독성**: Basic 때와 동일한 한계(표가 매우 클 경우 가독성 저하, 상한값으로만 방어).
- **다음 단계(Team)**: 로그인·다중사용자·공유·권한이 아직 전혀 없음 — Pro는 여전히 "내 컴퓨터에서 나 혼자 쓰는" 전제.
